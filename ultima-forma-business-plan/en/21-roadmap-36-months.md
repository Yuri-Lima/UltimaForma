# Roadmap 36 Months

## Phase 0 — Foundation (0--6 months)

### Technical Milestones
- Functional Orchestration Platform MVP
- Enterprise API* with basic verification flow
- Integration with at least 1 credential issuer (Bank or Telecom)
- Identity Wallet in minimal version (Android and iOS mobile)

### Open-Source Milestones
- Publish credential verification library on GitHub (Apache 2.0)
- Publish wallet SDK reference implementation
- Protocol specification draft v0.1
- Developer documentation portal (getting started, API reference)

### Commercial Milestones
- Product documentation and commercial material
- First conversations with ICPs* and value proposition validation

### Regulatory Milestones
- Mapping of LGPD* and AML*/KYC* requirements for the model
- Definition of terms of use and privacy policy

---

## Phase 1 — Pilot (6--12 months)

### Technical Milestones
- 1 credential issuer integrated in production
- Validated end-to-end flow (issuance → wallet → verification)
- Latency, availability, and security metrics monitored

### Open-Source Milestones
- Protocol specification v1.0
- Developer CLI tools released
- First external contributor guidelines published
- Developer sandbox environment live

### Commercial Milestones
- 1 anchor partner in active pilot
- 10 thousand verifications performed in the period

### Regulatory Milestones
- Legal opinion on regulatory positioning
- Documented consent and audit processes

---

## Phase 2 — Initial Scale (12--24 months)

### Technical Milestones
- 2--3 integrated issuers
- Performance and scalability improvements
- Support for qualified credentials (when applicable)

### Open-Source Milestones
- Developer community launch (Discord, regular meetups)
- Extended cryptographic libraries (selective disclosure)
- Third-party wallet implementations begin (using open SDK)
- Developer sandbox with full API access

### Commercial Milestones
- 5--10 enterprise clients in production
- 200--500 thousand verifications/year
- First subscription and enterprise SLA* contracts

### Regulatory Milestones
- Certifications or audits as required by the market
- Adaptation for regional requirements (LATAM*)

---

## Phase 3 — Expansion (24--36 months)

### Technical Milestones
- Preparation for multi-region (e.g., Brazil + 1 LATAM or EU* country)
- New credential types and use cases

### Open-Source Milestones
- Protocol governance formalization (community maintainers, foundation model evaluation)
- Ecosystem certification program for issuers and wallet providers
- 2,000+ GitHub stars, 10,000+ monthly SDK downloads

### Commercial Milestones
- Internationalization or expansion to 2nd vertical
- Preparation for Series A round
- 1--2 million verifications/year

### Regulatory Milestones
- Alignment with eIDAS* (if expansion to EU)
- Consolidated compliance processes

---

## Glossary (acronyms and terms)

- **AML**: Anti-Money Laundering; rules and controls to combat money laundering.
- **API**: Application Programming Interface; interface for integration between systems.
- **CLI**: Command-Line Interface; developer tool for terminal-based interaction.
- **eIDAS**: European regulation on electronic identification and trust services.
- **ICP**: Ideal Customer Profile; ideal customer profile.
- **KYC**: Know Your Customer; process of verifying client identity.
- **LATAM**: Latin America.
- **LGPD**: General Data Protection Law (Brazil).
- **SDK**: Software Development Kit; set of tools for building on a platform.
- **SLA**: Service Level Agreement; service level agreement.
- **UE**: European Union.
